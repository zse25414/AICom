### 產品背景：Lumina AI
Lumina AI 是一款個人與團隊生產力工具，目前以單一 HTML 檔案形式存在（Tailwind + Vanilla JS + localStorage），核心功能包含：
- 目標分解器、智能排程、AI 教練
- 團隊模式（群組代碼加入 + 主管指派任務 + 完成追蹤）

### 新模組：團隊專屬知識庫 AI（Advanced RAG Knowledge Base）

**目標**：
打造一個**真正專屬於團隊/專案的 AI 知識庫**。團隊可上傳工程文件、新人手冊、SOP、過往專案記錄、會議筆記、程式碼等資料，讓 AI 成為「讀過所有團隊資料的專屬智慧助手」。

核心價值：
- 新人快速融合（問 AI 就能得到準確、基於公司脈絡的答案）
- 減少重複勞動與知識流失
- 加速專案進展（AI 可根據知識庫內容提出建議、任務拆解、風險提醒）

**必須實現真正的 RAG 架構**（而非簡單文字注入）：
- 文件上傳 → 解析 → 切塊（chunking）→ Embedding → 存入向量資料庫
- 使用者查詢時 → 檢索最相關 chunk → 注入 LLM Prompt → 生成回答，並標註來源
- 支援混合檢索（keyword + semantic）

**核心功能需求**：
1. **知識庫管理**
   - 建立多個知識庫（例如：專案A、專案B、部門知識）
   - 上傳多種檔案（PDF、Word、Markdown、TXT、程式碼、圖片附 OCR）
   - 自動解析、切塊、嵌入
   - 支援文件版本管理與刪除

2. **AI 對話介面**
   - 在聊天中可選擇「使用 XXX 知識庫」
   - 支援 @知識庫 或特定指令
   - 回答必須引用來源文件（提升可信度）
   - 整合既有功能：AI 可根據知識庫內容自動幫忙「目標分解」或「智能排程建議」

3. **權限與安全**
   - 群組層級隔離（不同群組知識庫不互通）
   - 角色權限（管理員可上傳/管理，一般成員可查詢）
   - 資料加密與隱私保護（企業最重視）

4. **額外加分功能**
   - 自動摘要上傳文件
   - 知識圖譜視覺化（重要概念關聯）
   - 新文件上傳後主動通知相關成員
   - 與任務系統結合（AI 根據知識自動建議任務）

### 技術架構要求（Production Grade）

**推薦技術棧**（請提出最適合的方案）：
- **後端**：Python FastAPI / Node.js
- **RAG 框架**：LangChain 或 LlamaIndex（建議說明選擇理由）
- **向量資料庫**：Chroma（本地開發）、Pinecone / Weaviate / Qdrant（正式環境）
- **Embedding 模型**：OpenAI text-embedding-3、Voyage、或開源 BGE / Snowflake-arctic-embed
- **LLM**：支援 Grok、Claude、GPT-4o、或本地 Llama 3.1
- **前端**：與現有 Lumina AI HTML 整合（React / Vue 也可）
- **檔案解析**：Unstructured.io、LlamaParse 或 pdfplumber + OCR
- **部署**：Docker + Vercel / Railway / AWS

**階段性開發建議**：
- MVP：簡單 RAG（上傳 + 檢索 + 聊天）
- 進階：多知識庫、混合檢索、引用來源、與任務系統深度整合
- 企業版：細權限、審計日誌、On-Premise 部署

**設計原則**：
- 對使用者極致簡單（拖拉上傳即可）
- 回答必須可靠且可追溯
- 與 Lumina AI 既有 UI 風格一致（現代、暗色、流暢）
- 注重隱私與企業合規
